說說軟文-講講規矩-請記住軟的文

發佈於 2021-06-26

​「了解軟文，整理環境，提供工具，把控質量，提高效率。」

2018年

08月10月，設計軟的文平臺業務邏輯
10月12月，開發微信小程序
12月，開始測試平臺，歡迎使用

有人說好的開始就是成功的一半，可能我比較長，我的開始，觀望了四五年，才開始嘗試，從開始接觸軟文，到做編輯，然後被折磨，最後找到方法。

8月開始，公司項目逐漸收尾，月底就已經完工，領導也準備出差去總公司交工。趁著這段時間，開始設計「軟的文」平臺。

平臺搭建起來之後，就出現了很多問題，第一個問題是，很多人不了解軟文。

// 那你了解「軟文」嗎? //

很多人對軟文的理解就是「廣告」，這的確是很大的一方面，但是我覺得軟文是，帶有目的性和誘導的文章，當然小說不算。

現在很多行業都需要軟文來維護、運營，但是每天需要的文案是寫不完的，所以就需要找兼職寫手。只不過我看到的任務都在QQ群里和微信群里，豆瓣也很多。

對於軟文編輯來說，每項任務都需要這樣整理。但是這樣有很大的弊端，20人的任務還能勉強處理，那200人，上千人就很麻煩。50篇任務還能分配，500篇，上千篇就亂了。Excel的確強大，但是我是弱小的，況且整理數據的能力還在平均水平之下。

創建這個平臺的初衷，就是當初自己做編輯的時候，希望找到一個工具可以減少自己的工作量。但因為沒有平臺，只能自己記錄，哪位寫手負責哪篇任務，哪篇任務的審核進度，還有哪位寫手人間蒸發了。

所以沒有工具，只能自己開發。

// 幫我算一算 //

關於這個平臺，還要感謝當初的一位大學老師，ta是南方的，除了知道ta是老師，其他的就只有ta當時發布的任務。400字分成三段，首尾各150字，中間一段可以去參考網站複製，末尾配一張圖。

一篇任務，1.5元。

可能讀者會不屑，可能會覺得太浪費時間，不過有時候人們很容易就會忘記，積少成多的道理，當然這個道理不是針對寫手，而是針對那位老師。

這是一位大學計算機軟體專業的老師，ta自己開發了一個簡單的網站，可以在線領取任務，編寫任務，提交稿件，修改稿件。

ta的工作，只需要每天把關鍵詞上傳到網站上，然後分批次把寫手們提交的稿件，發到甲方。

如果一天100篇任務，抽取佣金。ta一個月可以得到多少稿費？

「1.5 × 100 × 30 = ???」

// 聊聊遊戲規則 //

每個平臺都有自己的規則，但是現在我沒有找到類似的平臺。所以，遊戲開始前，要講講「 軟的文 」的遊戲規則。

在做編輯寫任務的時候，我了解到的最多的一個問題，就是寫手和編輯互不信任，這是最難解決的一個問題。所以我希望從最開始，「 軟的文 」的用戶要有起碼的原則和底線。

各大APP收集用戶信息的話題，現在越來越熱。「 軟的文 」也需要收集用戶信息，這裡要說明原因。

微信登錄：微信授權，用於用戶登錄。

記錄手機號：綁定手機號，區別用戶，保證帳戶唯一性。

身份證信息：記錄身份證號碼，實名認證，進一步區別用戶，保證帳戶唯一性。

收集信息的作用有兩個。

第一個是為了區別用戶，保證帳戶唯一性，方便記錄寫手和編輯的歷史數據，方便查閱。

第二個是為了避免惡意破壞行為，平臺會根據用戶違約記錄，酌情封停用戶使用權限，一旦封停，永不解封，並且違約用戶綁定的信息，永久禁止再次綁定使用。

「 軟的文 」平臺的原則是尊重文案創作者，為打造一個誠實守信、有原則、有底線的平臺，不會姑息任何一次惡意破壞。

// 我說完了，該你了 //

現在平臺是完全免費的。

如果您想使用平臺，但是發現平臺功能不適合您的任務，請聯繫我，我會根據您的任務，專門制定功能模板。

如果您想參與平臺運營與開發，請聯繫我，沒有門檻，沒有限制，只需要您尊重文字，願意參與。
